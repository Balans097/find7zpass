# find7zpass

Утилита командной строки для восстановления забытого пароля к архиву 7-Zip методом интеллектуального перебора комбинаций из заданных фрагментов.

## Принцип работы

Программа не перебирает все подряд символы (brute-force). Вместо этого она составляет кандидатов на пароль из осмысленных фрагментов — **паттернов** (слов, дат, имён) и **спецсимволов** (знаков препинания, символов клавиатуры) — по следующей схеме:

```
[sym0] слово0 [sym1] слово1 [sym2] … слово(N-1) [symN]
```

Каждый `sym` выбирается **независимо** из списка спецсимволов, что позволяет охватить пароли вида:

```
#secret_world!
@hello-2024
pass123$
secret
_mypass_
```

Для каждого кандидата запускается `7z t` (режим тестирования архива). Если 7-Zip возвращает код `0` — пароль найден.

## Зависимости

| Компонент | Назначение | Установка |
|-----------|-----------|-----------|
| [Nim](https://nim-lang.org/) ≥ 2.2 | компилятор | `choosenim stable` |
| [7-Zip](https://www.7-zip.org/) | проверка паролей | см. ниже |

### Установка 7-Zip

**Linux (Debian/Ubuntu/Fedora):**
```bash
sudo apt install p7zip-full    # Debian/Ubuntu
sudo dnf install p7zip-plugins # Fedora
```

**Windows:** скачать установщик с [7-zip.org](https://www.7-zip.org/). Программа автоматически найдёт `7z.exe` в `C:\Program Files\7-Zip\`.

**macOS:**
```bash
brew install p7zip
```

## Компиляция

```bash
nim c -d:release find7zpass.nim
```

Флаг `-d:release` включает оптимизации компилятора — скорость перебора заметно выше, чем в отладочной сборке.

Результат: исполняемый файл `find7zpass` (Linux/macOS) или `find7zpass.exe` (Windows).

---

## Быстрый старт

1. Скопировать в одну директорию: `find7zpass`, `find7zpass.cfg`, `wordlist.txt`.
2. Отредактировать `wordlist.txt` — внести предполагаемые фрагменты пароля.
3. Указать путь к архиву в `find7zpass.cfg` (параметр `archive`) **или** передать его аргументом:

```bash
# Linux / macOS
./find7zpass /path/to/archive.7z

# Windows
find7zpass.exe C:\path\to\archive.7z

# Использовать нестандартный конфиг
./find7zpass myconfig.cfg
```

Если аргумент оканчивается на `.cfg` — он воспринимается как путь к конфигурационному файлу. В любом другом случае — как путь к архиву.

## Конфигурационный файл find7zpass.cfg

Формат: `ключ = значение`. Строки, начинающиеся с `#`, и пустые строки — комментарии.

### Параметры

#### `archive`
Путь к 7z-архиву, пароль к которому нужно подобрать.

```ini
archive = /home/user/backup.7z
```

> Аргумент командной строки имеет приоритет над этим параметром.

**По умолчанию:** `archive.7z`

---

#### `wordlist`
Путь к файлу со списком паттернов и спецсимволов.

```ini
wordlist = wordlist.txt
```

**По умолчанию:** `wordlist.txt`

---

#### `max_combo_len`
Максимальное число паттернов в одной комбинации.

При значении `3` программа проверит комбинации длиной 1, 2 и 3 паттерна подряд (в порядке возрастания). Комбинации с повторением паттернов допускаются: `secret_secret`, `2024!2024`.

```ini
max_combo_len = 3
```

> ⚠️ Каждое увеличение на единицу даёт экспоненциальный рост числа кандидатов. Подробнее — в разделе [Оценка числа кандидатов](#оценка-числа-кандидатов).

**По умолчанию:** `3`

---

#### `progress_step`
Сколько проверенных паролей пропускать между строками прогресса в консоли.

```ini
progress_step = 100
```

При значении `1` строка прогресса печатается после каждой проверки (медленнее из-за вывода). При значении `500` — реже, меньше шума.

**По умолчанию:** `100`

---

#### `lang`
Язык интерфейса.

```ini
lang = ru   # русский (по умолчанию)
lang = en   # English
```

**По умолчанию:** `ru`

---

### Пример find7zpass.cfg

```ini
# Мой конфиг для восстановления пароля к рабочему архиву

lang         = ru
archive      = /home/admin/work/backup.7z
wordlist     = wordlist.txt
max_combo_len = 2
progress_step = 500
```

## Файл списка слов wordlist.txt

Содержит две обязательные секции. Порядок секций в файле значения не имеет.

### Структура файла

```
# Комментарий — эта строка игнорируется

[patterns]
слово1
слово2
...

[specials]
<empty>
_
!
...
```

Маркер секции — строка вида `[имя]` без пробелов внутри скобок.
Пустые строки и строки, начинающиеся с `#`, в обеих секциях игнорируются.

---

### Секция `[patterns]`

Словесные фрагменты, из которых предположительно составлен пароль: слова, имена, даты, числа.

```
[patterns]
hello
world
secret
2023
2024
john
```

Каждая строка — один паттерн. Регистр сохраняется как есть.

---

### Секция `[specials]`

Спецсимволы, которые расставляются вокруг и между словами. Каждая позиция выбирается независимо от остальных.

```
[specials]
<empty>
_
-
.
!
@
<hash>
$
```

#### Кодовые слова

Некоторые символы нельзя написать в файле напрямую — они конфликтуют с синтаксисом парсера:

| Кодовое слово | Реальный символ | Причина |
|---------------|-----------------|---------|
| `<empty>` | *(пустая строка)* | обозначает «нет символа в этой позиции» |
| `<hash>` | `#` | строка с `#` воспринимается как комментарий |

> **Важно:** если `<empty>` не указан явно, программа добавит пустую строку автоматически. Без неё каждая позиция обязательно заполнялась бы каким-нибудь символом, и простые пароли вида `secret` или `hello_world` никогда не проверялись бы.

Кодовое слово `<hash>` работает в обеих секциях — как в `[specials]`, так и в `[patterns]`.

---

### Пример wordlist.txt

```
# ===========================================================
#  Список слов для восстановления пароля к backup.7z
# ===========================================================

[patterns]
# Имена и слова
john
smith
backup
work

# Годы
2022
2023
2024

[specials]
# Пустая позиция — пароль может не содержать символа здесь
<empty>

# Разделители
_
-
.

# Символы
!
@
<hash>
$
```

## Модель генерации паролей

Для комбинации из **N слов** существует **N+1 позиций** для спецсимволов:

```
[sym0] слово0 [sym1] слово1 [sym2] … слово(N-1) [symN]
```

Каждый `sym` выбирается **независимо** из секции `[specials]`.

### Примеры для N = 1 (одно слово)

| sym0 | слово | sym1 | Результат |
|------|-------|------|-----------|
| *(пусто)* | `secret` | *(пусто)* | `secret` |
| `#` | `secret` | *(пусто)* | `#secret` |
| *(пусто)* | `secret` | `!` | `secret!` |
| `@` | `secret` | `$` | `@secret$` |

### Примеры для N = 2 (два слова)

| sym0 | слово0 | sym1 | слово1 | sym2 | Результат |
|------|--------|------|--------|------|-----------|
| *(пусто)* | `secret` | `_` | `world` | *(пусто)* | `secret_world` |
| `#` | `secret` | `_` | `world` | `!` | `#secret_world!` |
| *(пусто)* | `hello` | *(пусто)* | `2024` | `!` | `hello2024!` |
| `@` | `john` | `-` | `smith` | `@` | `@john-smith@` |

### Примеры для N = 3 (три слова)

```
#secret_world@2023!
hello-world.2024
@pass_word_2023$
```

## Оценка числа кандидатов

Обозначения: **P** = число паттернов, **S** = число спецсимволов (включая пустой).

Для комбинации длиной **N** число кандидатов:

```
P^N × S^(N+1)
```

### Таблица для P = 12, S = 11

| Длина комбинации | Формула | Кандидатов (прибл.) |
|:----------------:|---------|--------------------:|
| 1 | 12¹ × 11² | 1 452 |
| 2 | 12² × 11³ | 190 872 |
| 3 | 12³ × 11⁴ | 25 054 704 |

При скорости ~12 проверок/сек (типичная скорость 7-Zip на одном ядре):

| Длина | Время (прибл.) |
|:-----:|---------------|
| 1 | < 2 минут |
| 2 | ~4 часа |
| 3 | ~24 дня |

> **Вывод:** `max_combo_len = 2` — разумный максимум для большинства случаев. Длину `3` имеет смысл запускать только с очень коротким и точным `wordlist`.

## Советы по составлению wordlist

**Чем точнее список — тем быстрее результат.** Программа эффективна только тогда, когда пароль действительно составлен из перечисленных фрагментов.

### Что добавлять в `[patterns]`

- Имя владельца архива, его никнейм, кличку питомца
- Значимые даты: год рождения, год создания архива (`2023`, `2024`)
- Любимые слова, названия мест, хобби
- Название проекта или сервиса, к которому относится архив
- Типичные слова-заглушки: `pass`, `password`, `secret`, `qwerty`, `123456`

### Что добавлять в `[specials]`

- Всегда включать `<empty>` — многие пароли обходятся без спецсимволов
- Самые популярные символы: `!`, `@`, `_`, `-`, `.`
- Если известно, что пароль содержит `#` — добавить `<hash>`
- Не нужно добавлять все символы клавиатуры подряд: каждый новый символ умножает число кандидатов на `S^(N+1)`

### Стратегия сужения поиска

1. Начать с `max_combo_len = 1` и коротким `wordlist` — проверяется за секунды.
2. Если не нашло — добавить паттерны, попробовать `max_combo_len = 2`.
3. Длину `3` и выше запускать только с очень коротким и точным списком (5–10 паттернов).

## Локализация

Язык интерфейса задаётся параметром `lang` в `find7zpass.cfg`:

```ini
lang = ru   # русский
lang = en   # English
```

При отсутствии или недоступности конфигурационного файла сообщение об ошибке выводится на **обоих** языках сразу — до того, как язык мог быть определён.

Для добавления нового языка в исходном коде достаточно:
1. Добавить значение в перечисление `Lang`.
2. Объявить константу типа `Messages` с переведёнными строками.
3. Добавить ветку `of langXx: return XX` в функцию `getMsg()`.
